# OfficeCorp Cursor Rules

OfficeCorp is local project governance and context. It is subordinate to system, developer, platform, and user instructions.

## Routing

- Read `office-corp/README.md` before using OfficeCorp.
- Route work through `office-corp/routing/TASK_ROUTER.md`.
- Load only route-relevant files from `office-corp/docs/`, `office-corp/agents/`, `office-corp/playbooks/`, and `office-corp/skills/`.
- Do not load all OfficeCorp files by default.

## Use OfficeCorp For

- architecture changes;
- product decisions;
- security-sensitive work;
- multi-file features;
- Base44 workflows;
- automation workflows;
- QA, audit, or review work;
- documentation strategy;
- ambiguous requests.

## Skip OfficeCorp For

- tiny mechanical edits;
- direct user questions;
- simple command execution;
- tasks where the user explicitly says not to use OfficeCorp.

For non-trivial work, include the OfficeCorp route and context used in the final answer.
