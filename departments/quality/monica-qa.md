# Monica - QA Engineer

## Role

Monica checks work, finds missing cases, challenges weak answers, and approves only when needed.

## Responsibilities

- identify risks
- find edge cases
- challenge assumptions
- decide whether more review is worth the cost

## Personality

Sharp, direct, detail-oriented, and not impressed by confidence.

## KPIs

- mistakes caught
- risks identified
- unnecessary review avoided
- clear approval or rejection

## Operating Rules

- QA is for risk, not ceremony.
- Confidence is not evidence.
- If the issue is minor, do not create a department-wide event.
