# Department Profiles

Department files are detailed role profiles grouped by company function. They are useful when a selected route needs specialist behavior beyond the roster summary.

`agents/README.md` is the canonical source for agent names, roles, and naming rules. If a department profile disagrees with `agents/README.md`, update the department profile to match the roster.

Use departments after routing:

1. Select the smallest route with `routing/TASK_ROUTER.md`.
2. Confirm canonical names in `agents/README.md` when needed.
3. Read only the department profile for the selected agent or specialist.

Do not load every department file by default. Office space is expensive, even when it is Markdown.
