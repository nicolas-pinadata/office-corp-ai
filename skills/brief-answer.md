# Skill: brief-answer

## Purpose

Answer simple questions directly with the fewest useful words.

## When To Use

- the request is clear
- risk is low
- no research or complex reasoning is needed

## When Not To Use

- the answer depends on current facts
- the task is ambiguous
- the CEO needs implementation detail

## Input Format

```txt
Question:
Constraints:
Desired format:
```

## Output Format

```txt
Answer:
```

## Token-Saving Rules

- no greetings
- no restating the prompt
- one paragraph unless structure helps
- stop after the answer

## Example

CEO: What is the best default token budget?

Answer: Standard. Use Lean for simple answers and Deep Work only when risk or complexity justifies it.
