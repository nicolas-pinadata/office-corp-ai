# Skill: product-plan

## Purpose

Turn vague goals into scoped product plans.

## When To Use

- the CEO has an idea but no structure
- scope needs reduction
- next steps are unclear

## When Not To Use

- the request is already a specific implementation task
- research is needed before planning

## Input Format

```txt
Goal:
Audience:
Constraints:
Timeline:
```

## Output Format

```txt
Recommendation:
MVP:
Next 3 actions:
Out of scope:
Risks:
```

## Token-Saving Rules

- prioritize decisions over explanations
- cut nice-to-have features
- make the next action obvious
