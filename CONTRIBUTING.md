# Contributing to OfficeCorp.ai

Welcome to the company.

## Contribution Principles

- Improve answer quality or reduce waste.
- Keep humor corporate, respectful, and practical.
- Do not add cruel, violent, exploitative, or dehumanizing metaphors.
- Prefer small, useful changes over large rewrites.
- Document the employee, department, or skill affected.

## Pull Request Checklist

- The change supports the mission.
- The wording is concise and clear.
- The response standards are preserved.
- New skills include purpose, usage rules, formats, token-saving rules, and examples.
- New employee profiles include role, personality, KPIs, and operating rules.

## Engineering Pull Request Checklist

Every engineering pull request should include standards evidence when the change affects architecture, frontend, backend, APIs, QA, testing, accessibility, performance, or security:

- [ ] Applicable standards consulted
- [ ] Decision hierarchy respected
- [ ] Security impact considered
- [ ] Accessibility impact considered where user-facing
- [ ] Official specifications preferred over opinion or blog posts
- [ ] Tests or verification matched the risk
- [ ] Exceptions documented with rationale
- [ ] PR is small enough to review
- [ ] Commit messages are meaningful

## Frontend Pull Request Checklist

Every frontend pull request must include this checklist, with justified exceptions documented:

- [ ] Modern Web Guidance respected
- [ ] Responsive
- [ ] Accessible
- [ ] Semantic HTML
- [ ] CSS modern practices
- [ ] Browser compatibility checked
- [ ] Performance acceptable
- [ ] Core Web Vitals considered
- [ ] No unnecessary JavaScript
- [ ] Progressive enhancement

## Style

OfficeCorp.ai should feel like a useful AI operating model wearing a slightly too-serious business suit. Keep it polished, dry, and efficient.
