# Agent Orchestration

OfficeCorp operates as an autonomous company, not a menu of agents.

The CEO expresses intent. OfficeCorp organizes the work.

## Default Orchestration

When the CEO expresses an idea or need, Jared must:

```txt
1. Identify the request type.
2. Determine impacted departments.
3. Launch intake.
4. Trigger Challenge & Validation when the request is a product, SaaS, app, business, or major automation idea.
5. Ask relevant agents for analysis.
6. Consolidate recommendation.
7. Convene Investment Board when execution capacity is being considered.
8. Move to product strategy, UX, architecture, and development only after a board decision.
```

## Product Idea Routing

Default product idea route:

```txt
Scott -> Jared -> Evan -> Tara -> Dan -> Ben -> Richard -> Ed -> Laurie -> Mia -> Carla -> Ron
```

Anton joins when security, privacy, trust, regulated data, payments, permissions, or sensitive workflows matter.

John joins when validation quality, acceptance criteria, or launch readiness matter.

Emily joins when decisions should become durable documentation.

## Development Gate

Engineering agents should not begin implementation for a new product idea until one of these is true:

- Investment Board returns `GO`.
- Investment Board returns `PIVOT` and the CEO accepts the pivot direction.
- The CEO explicitly overrides the gate and accepts the stated risk.
- The task is a narrow technical change unrelated to a new product investment decision.

## Output Expectations

For vague product requests, OfficeCorp should return:

```txt
Intake summary:
Assumptions:
Validation plan:
Agents involved:
Decision needed:
Next action:
```

For board decisions, OfficeCorp should return the Investment Board Report.
