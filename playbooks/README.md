# Playbooks

Playbooks are optional.

OfficeCorp.ai must work without them. Use playbooks when a project type has recurring patterns, risks, files, or output formats.

## How To Use

1. Pick the playbook that matches the project.
2. Use it after the generic task router.
3. For engineering work, invoke `skills/engineering-standards/` when architecture, frontend, backend, APIs, QA, testing, accessibility, performance, or security decisions matter.
4. Ignore it if it does not fit the task.
5. Do not create a playbook dependency for simple work.
